# Add to Backlog

Add this to the project backlog: **$ARGUMENTS**

## Steps

1. **Read current backlog** at `docs/BACKLOG.md`

2. **Determine category**:
   - **Ideas**: Rough concepts, "someday" features
   - **Planned**: Agreed upon, will implement soon
   - **Bugs**: Known issues to fix

3. **Add the item** to the appropriate section with:
   - Clear, concise description
   - `[ ]` checkbox format
   - Any relevant context

4. **Update "Last Updated" date** at the top of the file

## Example Format

```markdown
## Ideas
- [ ] **Feature name** - Brief description of what it does
```

## Notes

- Keep descriptions short but clear
- Add context if the idea might be forgotten
- Don't add duplicates - check if similar item exists
